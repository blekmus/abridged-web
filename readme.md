# Abridged Web

## Development

Run the frontend dev server and backend seperately in two different terminals:

```sh
bun run dev
```
```sh
go run ./main.go
```

## Build

Build manually:

```sh
bun run build
APP_ENV=production go build -o abridged ./main.go
```

## Runtime

Run the application using the following command. Environment variables are read at runtime from the process environment.

Environment variables:

- `PORT`: server port, defaults to `8080`
- `ARCHIVE_ROOT`: archive directory, defaults to `../archive`

Example:

```sh
PORT=3000 ARCHIVE_ROOT=/path/to/archive ./abridged
```

## Features

### Video Metadata Cache

On startup, the backend requires `ARCHIVE_ROOT` to exist and reads video descriptions and durations from `ARCHIVE_ROOT/.abridged-video-metadata.json`. That file must exist before the server starts and is generated by the standalone metadata CLI.

The web app does not run `ffmpeg` or `ffprobe`. During catalog scanning:

- matching cache entries are used when `size` and `modTimeUnixNano` match the video file
- stale cache entries produce empty metadata for that video
- the cache file is never modified by the web app
